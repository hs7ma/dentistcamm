📋 نظرة عامة

DentistCam نظام متكامل لتصوير الأسنان عن بُعد، يجمع بين كاميرا ESP32-CAM وواجهة ويب حديثة وتحليل طبي آلي عبر الذكاء الاصطناعي. يهدف النظام لتقديم تشخيص مساعد سريع للأطباء، الطلاب، والمرضى في المناطق النائية.

⚙️ المكونات الفنية

1. ESP32-CAM (الفيرموير)

  • اللوحة: AI-Thinker ESP32-CAM
  • المكتبات: ESP32 core 3.x by Espressif (مدمجة)
  • الكاميرا: OV2640 — جودة JPEG قابلة للضبط (4-40)
  • IP ثابت: 172.20.10.10 يُضبط قبل WiFi.begin (بدون DHCP)
  • الفلاش: LED على GPIO 4 — ضبط 0-255 عبر LEDC PWM

هيكل الـ Prompt

أنت مساعد طبيب أسنان. حلّل هذه الصورة السنية بإيجاز:

**الملاحظات:** (نخر، التهاب لثة، جير، تآكل، تلوّن — مع الموقع)
**التوصية:** (إجراء مقترح لكل مشكلة)
**الأولوية:** طبيعي / يحتاج متابعة / يحتاج علاج / يحتاج تدخل عاجل

كن موجزاً. هذا تحليل مساعد وليس تشخيصاً نهائياً.

🔄 تدفّق العمل (Workflow)

سيناريو التحليل الكامل

  1. المستخدم يضغط زر "تحليل بالذكاء" على الواجهة
  2. الواجهة ترسل GET http://172.20.10.10/capture لـ ESP32
  3. ESP32 يلتقط Frame عالي الجودة من OV2640 ويرجعه كـ JPEG
  4. المتصفح يحوّل الـ Blob إلى Base64 Data URL عبر FileReader
  5. Data URL يُرسل لـ OpenAI: POST /v1/chat/completions
  6. GPT-4o-mini يحلّل الصورة ويرجع تقريراً بالعربية
  7. الواجهة تعرض التقرير مع تمييز الأولوية بلون مناسب

تدفق الإقلاع

  1. iPhone Hotspot يُفعّل (مع Maximize Compatibility)
  2. الحاسوب يتصل بالـ Hotspot
  3. ESP32 يُشغَّل → يضبط IP 172.20.10.10 قبل WiFi.begin
  4. ESP32 يبدأ خوادم HTTP على المنفذين 80 و 81
  5. المستخدم يفتح run.bat → الواجهة على localhost:5173
  6. الواجهة تجلب IP الكاميرا من .env أو localStorage
  7. health check polling يبدأ كل 3 ثوانٍ → يتحقق من اتصال ESP32
  8. عند نجاح الاتصال — <img> يبدأ MJPEG stream تلقائياً

🔒 الأمان والخصوصية

مفتاح OpenAI

  • محفوظ في frontend/.env ويُحقن وقت البناء عبر import.meta.env
  • الملف مُتجاهَل من Git عبر .gitignore
  • المفتاح يظهر في حزمة JS النهائية (Vite يبني client-side فقط)
تنبيه: هذه الواجهة client-side — أي مستخدم يفتح DevTools يستطيع رؤية المفتاح. النظام مصمّم للاستخدام الشخصي/التعليمي. للإنتاج العام يجب إضافة backend proxy.

WiFi credentials

  • محفوظة مباشرة في أعلى ملف esp32_dental_cam.ino
  • عدّل قيم WIFI_SSID و WIFI_PASSWORD قبل الرفع

البيانات المرسلة لـ OpenAI

  • الصورة فقط — لا معلومات شخصية
  • OpenAI لا تُخزّن صور API بشكل افتراضي
  • الاتصال HTTPS مُشفّر

⚖️ نقاط القوة والقيود

نقاط القوة

  • سهولة النقل: ملف واحد run.bat ينصّب ويشغّل
  • IP ثابت مضمون: لا حاجة لإعادة ضبط الواجهة كل مرة
  • كلفة منخفضة: ~0.01 سنت لكل تحليل
  • دعم عربي كامل: RTL، تقارير عربية، رسائل خطأ عربية
  • لا backend: كل شيء client-side (لا server يحتاج صيانة)
  • يعمل بدون شبكة منزلية: iPhone Hotspot كافٍ

القيود الحالية

  • مفتاح OpenAI مكشوف: client-side bundle — للاستخدام الشخصي فقط
  • iPhone Hotspot فقط: الكود مخصص لـ subnet /28
  • لا تخزين سحابي: الصور تُحفظ محلياً فقط
  • لا قاعدة بيانات: التقارير لا تُحفظ بين الجلسات
  • تحليل واحد في وقت واحد: لا queue للطلبات
  • iPhone يُغلق Hotspot: بعد 90 ثانية بدون أجهزة متصلة
